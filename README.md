## 1. 学生学时明细页（老师端）

**归属章节**：§5.4 老师端功能设计

### 1.1 页面位置

老师端新增页面：**学生学时明细页**。入口来自课程学生列表，点击某个学生进入。

### 1.2 页面内容

以单个学生为维度，展示该生本学期**所有学时来源**的聚合视图：

| 来源类型               | 说明                          |
| ---------------------- | ----------------------------- |
| 学生自提交（课程相关） | 学生主动提交的 A 类打卡       |
| 学生自提交（其他运动） | 学生主动提交的 B 类打卡       |
| 老师任务完成           | 完成老师发布的课程任务        |
| 校队抵扣               | 校队成员身份自动/手动抵扣     |
| 社团抵扣               | 体育社团成员身份自动/手动抵扣 |
| 老师手动加抵           | 老师手动输入的抵扣记录        |

每条记录显示：来源类型、运动类型、申请小时、审批小时、提交时间、审核状态、审核人。

### 1.3 老师手动加抵卡

在学时明细页提供「手动加抵卡」操作入口：

| 字段       | 说明                |
| ---------- | ------------------- |
| 抵扣类型   | 课程相关 / 其他运动 |
| 抵扣小时数 | 手动输入            |
| 原因说明   | 文字备注（必填）    |
| 凭证上传   | 可选，支持图片      |

操作记录：标注来源为"老师手动加抵"，操作人、时间、原因全部留痕，纳入操作日志。

## 2. 校队/社团抵扣审核（老师端）

**归属章节**：§5.4 老师端功能设计

### 2.1 抵扣审核流程

当前设计为"系统自动抵扣"，改为以下流程：

```
管理员导入校队/社团成员名单
  → 系统生成抵扣记录（状态：待老师确认）
    → 任课老师在「学生学时明细页」或「待确认列表」中审核
        ├── 确认生效 → 抵扣正式计入学生总学时
        ├── 驳回 → 填写原因，学生端可见
        └── 修改小时数 → 调整后确认
```

### 2.2 老师待确认列表

老师端课程工作台新增一个待处理项：

> **待确认的抵扣记录：N 条**

点击进入列表，按学生展示所有"待老师确认"的校队/社团抵扣记录。

### 2.3 审核操作

| 操作       | 说明                                                          |
| ---------- | ------------------------------------------------------------- |
| 确认生效   | 该笔抵扣正式计入                                              |
| 驳回       | 附驳回原因，抵扣不生效，学生端可见反馈                        |
| 修改并确认 | 调整抵扣小时数后确认（例如校队训练实际出勤不足，减为 5 小时） |

### 2.4 与自动抵扣的关系

管理员可在管理端配置开关：

| 配置项       | 选项                  | 说明             |
| ------------ | --------------------- | ---------------- |
| 校队抵扣模式 | 自动生效 / 需老师确认 | 默认"需老师确认" |
| 社团抵扣模式 | 自动生效 / 需老师确认 | 默认"需老师确认" |

试点阶段走"需老师确认"，避免争议。成熟后可切换为"自动生效"。

## 3. 800/1000米成绩自动换算

**归属章节**：§6.6 体测成绩换算

### 3.1 换算规则

国家体测标准按年级组分两套标准，每 1 秒一个分数，不可有空档。

| 年级组    | 适用学生                               |
| --------- | -------------------------------------- |
| 大一/大二 | 入学年份为当前年份及上一年的本科生     |
| 大三/大四 | 入学年份为当前年份减 2 及减 3 的本科生 |

每个年级组 × 性别（男/女）需要独立维护一张换算表。

### 3.2 换算表配置（管理端）

管理员在管理端维护换算表。每条记录：

| 字段          | 说明                      |
| ------------- | ------------------------- |
| 年级组        | 大一/大二、大三/大四      |
| 性别          | 男 / 女                   |
| 项目          | 800米（女）/ 1000米（男） |
| 成绩（分:秒） | 原始跑步时间              |
| 对应分数      | 该秒数对应的体测分        |

**约束**：从满分到 0 分的每一秒都必须有对应分数，系统检查无缝覆盖，有空档不允许保存。

例如（大三/大四 · 男 · 1000米）：

| 成绩   | 分数 |
| ------ | ---- |
| 3'17"  | 100  |
| 3'18"  | 99   |
| 3'19"  | 98   |
| …     | …   |
| 6'00"+ | 0    |

### 3.3 自动识别逻辑

老师录入体测原始成绩时，系统自动：

1. 从学生信息获取**学号** → 提取入学年份 → 计算年级 → 匹配年级组（大一/大二 or 大三/大四）
2. 从学生信息获取**性别**
3. 匹配对应换算表
4. 输入原始成绩 → 即时返回换算分数

计算公式：

```
年级组 = (当前学期年份 - 入学年份) ≤ 2 ? "大一/大二" : "大三/大四"
换算表 = 查询(年级组 + 性别 + 项目)
分数 = 换算表.lookup(原始成绩)
```

### 3.4 数据库变更

**体测换算规则表（新增/变更）**：

| 字段            | 类型   | 说明                                |
| --------------- | ------ | ----------------------------------- |
| rule_id         | string | 规则 ID（主键）                     |
| grade_group     | enum   | 大一/大二、大三/大四                |
| gender          | enum   | 男 / 女                             |
| item_name       | string | 800米 / 1000米 / 跳远 / 仰卧起坐 … |
| raw_value       | string | 原始成绩（跑步为分:秒格式）         |
| converted_score | float  | 换算分数                            |
| version         | int    | 规则版本号                          |

## 4. 800/1000米免测申请

**归属章节**：§5.3 学生端功能设计（新增小节）、§6.6 体测成绩换算（新增小节）

### 4.1 流程

```
学生提交免测申请 → 上传证明文件 → 老师审核 → 通过/驳回
  ↓ 通过
该学生该项目体测成绩标记为「免测」，不参与该项计分（或按学校规则给基础分）
```

### 4.2 学生端：免测申请表单

| 字段     | 说明                              |
| -------- | --------------------------------- |
| 免测项目 | 默认 800/1000米（可扩展其他项目） |
| 申请理由 | 文字描述                          |
| 证明文件 | 医院证明 / 诊断书等，支持多张图片 |
| 提交时间 | 自动记录                          |

学生可查看申请状态：待审核 / 已通过 / 已驳回。

### 4.3 老师端：免测申请审核

老师端体测页面新增「免测申请」标签页：

| 显示内容       | 说明                                |
| -------------- | ----------------------------------- |
| 学生姓名、学号 | 基本信息                            |
| 免测项目       | 申请免除的项目                      |
| 申请理由       | 学生填写                            |
| 证明文件       | 可查看/下载                         |
| 审核操作       | 通过 / 驳回（附原因）/ 要求补充材料 |

### 4.4 通过后的成绩处理

| 处理方式   | 说明                                                             |
| ---------- | ---------------------------------------------------------------- |
| 成绩标记   | 该项目标记为"免测"，不计入该项目分数                             |
| 总成绩计算 | 免测项目不拖累总分（按学校实际规则，可能给基础分或排除该项权重） |
| 可配置     | 管理端可配置免测项目的计分策略（基础分 / 不计分 / 按比例折算）   |

### 4.5 数据模型

**免测申请表 ExemptionRequest（新增）**：

| 字段           | 类型     | 说明                              |
| -------------- | -------- | --------------------------------- |
| exemption_id   | string   | 申请 ID（主键）                   |
| student_id     | string   | 学生 ID                           |
| course_id      | string   | 课程 ID                           |
| item_name      | string   | 免测项目名称                      |
| reason         | string   | 申请理由                          |
| proof_files    | array    | 证明文件 URL 列表                 |
| status         | enum     | 待审核 / 已通过 / 已驳回 / 需补充 |
| reviewer_id    | string   | 审核人 ID                         |
| review_comment | string   | 审核意见                          |
| created_at     | datetime | 申请时间                          |
| reviewed_at    | datetime | 审核时间                          |

## 5. 任课老师查看校队/社团身份

**归属章节**：§5.4 老师端功能设计

### 5.1 学生组织身份查看

在学生学时明细页中，新增「组织身份」信息区域，展示该生所属的校队/社团：

| 信息项     | 说明                     |
| ---------- | ------------------------ |
| 组织类型   | 校队 / 社团              |
| 组织名称   | 具体队伍或社团名称       |
| 是否体育类 | 仅社团有此标识           |
| 身份状态   | 有效 / 无效 / 待确认     |
| 有效期     | 起止时间                 |
| 抵扣状态   | 已抵扣 / 待确认 / 已驳回 |
| 认证人     | 校队/社团负责人或管理员  |

### 5.2 老师操作

| 操作       | 说明                                       |
| ---------- | ------------------------------------------ |
| 标记已确认 | 确认该生组织身份信息无误                   |
| 标记存疑   | 对该生组织身份有疑问，进入管理员待处理列表 |
| 查看证明   | 查看该生校队/社团的成员证明或花名册记录    |

### 5.3 权限设计

- 老师只能**查看**和**确认/存疑**，不能修改校队/社团花名册
- 存疑记录由管理员与校队/社团负责人核实后处理
- 操作留痕，纳入操作日志
